## ARPA-H Selection Model

**Critical insight:** ARPA-H uses a **two-stage funnel**, not winner-take-all at Solution Summary.

### Two-Stage Process

| Stage | Input | Output | Our Focus |
|-------|-------|--------|-----------|
| **Solution Summary → Full Proposal** | ~30-40 serious applicants | ~10-15 invited | **Current goal** |
| **Full Proposal → Final Selection** | ~10-15 proposals | 5 finalists | Competition intensifies |
| **Year 1 → Year 2+** | 5 teams | Remove underperformers | Execution matters |

### What This Means

1. **Solution Summary is a wider funnel** - ARPA-H wants to see more detail before narrowing
2. **Full Proposal is the real competition** - Where the 5 finalists are chosen
3. **Year 1 performance matters** - Low performers removed, so selection isn't final

This means: A strong-but-not-perfect applicant with demonstrable capability has good chances at Solution Summary stage.

## FDA Speed Run + Open-Source Compatibility

**Question:** If rejected by ARPA-H and we self-fund FDA approval, does open-sourcing Popper give away our regulatory investment?

**Answer:** No. MDDT qualification attaches to the PRODUCT, not the code.

### Understanding MDDT Qualification

From the [FDA speed run docs](../0B-fda-speed-run/02-mddt-qualification-guide.md):

| Attribute | Description |
|-----------|-------------|
| **What we submit** | Specific version of Popper + validation studies + clinical evidence |
| **What FDA qualifies** | The PRODUCT (Popper with that evidence package) |
| **What's on GitHub** | The CODE (Apache 2.0 license) |

**Key insight:**

```
Open-source Popper (Apache 2.0)     ≠     Popper MDDT-Qualified™
├── Code on GitHub                        ├── Specific version
├── Anyone can fork                       ├── YOUR validation studies ($300K-1M)
├── NOT FDA-qualified                     ├── YOUR clinical evidence
├── Free                                  ├── YOUR FDA submission
└── No regulatory status                  ├── YOUR trademark
                                          └── Commercial product
```

### Why This Works

**If a competitor forks Popper:**

| They Get | They DON'T Get |
|----------|----------------|
| Code (Apache 2.0) | "Popper" name (trademark) |
| Architecture | YOUR validation data |
| Protocol implementation | YOUR clinical evidence |
| | YOUR MDDT qualification |
| | Shortcut to FDA approval |

**What they'd need to do:**

1. **Rename** (can't use "Popper" trademark) → "TheirPopper"
2. **Run their OWN validation studies** → $300K-1M + 12-24 months
3. **Submit their OWN MDDT proposal** → FDA review cycles
4. **Build their OWN clinical evidence** → Additional time + cost

**The moat isn't the code. It's the qualification.**

### Certification vs. MDDT Qualification

| Attribute | Hermes Certified™ | Popper MDDT-Qualified™ |
|-----------|-------------------|------------------------|
| **What it is** | Voluntary compliance program | FDA regulatory status |
| **Who grants it** | Us (private) | FDA (government) |
| **Value without ARPA-H** | Low (no mandate) | High (regulatory requirement) |
| **Cost to achieve** | ~$50K | ~$300K-1M |
| **Competitor barrier** | Low (just fork + rename) | High (must do own qualification) |

**Recommendation:** If rejected and pursuing FDA speed run:
- **Deprioritize** Hermes Certified program
- **Prioritize** Popper MDDT qualification
